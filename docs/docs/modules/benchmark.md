# 蚂蚁集团数据检索基准数据集指南

对于 Text2SQL 任务，我们提供数据集基准测试功能。它评估 Text2SQL 上的不同大型语言模型 (LLM) 和代理，涵盖语法正确性、语义准确性和执行有效性。输出可执行率、准确率等指标，并提供评估报告。

1. Ant Group开源的Text2SQL数据集存储库：[Falcon](https://github.com/eosphoros-ai/Falcon)
2. DB-GPT支持基于Falcon基准数据集的LLM评估

# 简介

为了客观公正地评估 Text2SQL 任务上的模型，我们提供了基准测试模块和数据集。该模块支持对DB-GPT框架中的所有模型进行综合评估，并提供评估报告。

该模块使用的基准数据集 [Falcon](https://github.com/eosphoros-ai/Falcon) 是来自 Ant Group 的高质量且不断发展的开源 Text2SQL 数据集。
该数据集旨在在复杂的跨域分析场景中对模型进行压力测试，重点关注：
 - SQL 计算挑战 — 多表连接、嵌套 CTE、窗口函数、排名、类型转换、正则表达式过滤器...
 - 语言挑战——中文模糊时间表达、口语商业术语、省略号、多意图问题……

> 该基准包括 28 个数据集和 90 个表格。截至目前，500道不同难度的中文题已正式发布。
> 
> 其中：简单：151，中：130，困难：219。

## 基准数据集的核心特征
- ✅ 多维度评估：语法正确性、语义准确性、执行有效性三层检查
- 🧠 动态难度级别：来自 Kaggle 数据集的 500 道中文问题（各种难度），涵盖多步推理、复杂嵌套查询和高级 SQL 功能
- ✍️详细的模式注释：丰富的模式信息，包括数据类型、自然语言别名、表关系和示例数据，帮助模型理解数据库结构
- 🌐真实场景建模：从蚂蚁集团真实生产场景中收集更多模糊的语言表达和问题（准备中）


# 系统设计
基准测试模块的核心功能：
- Text2SQL评估API：提供创建评估任务的API
- 基准执行框架：根据基准问题运行Text2SQL任务
- 结果比较框架：比较标准答案和LLM生成的SQL之间的结果，并汇总评估结果
- 数据集安装和数据库映射：安装基准数据集并将数据映射到数据库中，提供LLM SQL查询服务

# 评估指标

|公制|公式|描述 |
|--------------------------------|------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|执行率|语法正确的样本数/样本总数 |模型生成的 SQL 语句中语法正确且能够在数据库中正确执行的比例 |
|准确率 |语义正确的样本数量/样本总数 |模型生成的 SQL 语句中语法正确、在数据库中正确执行且语义正确的比例 |

# 如何使用

## 环境设置
- Step1：升级到V0.7.4并升级元数据数据库

    对于 SQLite，表架构默认会自动升级。对于MySQL，您需要手动运行DDL。文件 asset/schema/dbgpt.sql 包含当前版本的完整 DDL。可以在 asset/schema/upgrade 下找到特定于版本的 DDL 更改。例如，如果要从 v0.7.1 升级到 v0.7.4，则可以运行以下 DDL：
    ```
    mysql -h127.0.0.1 -uroot -p{your_password} < assets/schema/upgrade/v0_7_4/upgrade_to_v0.7.4.sql
    ```
- Step2：启动DB-GPT服务，等待基准数据集自动加载。当您看到日志行时，数据集已完成加载（大约 1~5 分钟）。

<p对齐=“左”>
  <img src={'/img/module/benchmark/env_load.png'} width="1000px"/>
</p>

- Step3：在DB-GPT平台注册LLM 
  - 方法1：通过配置文件配置。参考：【ProxyModel配置】(http://docs.dbgpt.cn/docs/next/installation/advanced_usage/More_proxyllms)
  - 方法2：通过产品页面配置。参考：[模型](http://docs.dbgpt.cn/docs/next/application/llms)

## 创建评估任务
- Step1：点击“创建基准”创建评估任务
- Step2: 输入任务名称并选择模型列表
- 第三步：提交任务

## 查看评估结果
- 当状态为“已完成”时，点击“查看详情”即可查看评估报告
- 报告显示：
  - 模型总数、问题数量、正确、错误和失败问题的数量
  - 对于每轮和模型：执行、正确、错误和失败问题的数量；可执行率；准确率
  - 可执行率和准确率的条形图

> 正确：模型正确回答了问题。错误：模型生成的 SQL 语法正确，但语义错误。失败：通常 SQL 在语法或语义上是错误的。
