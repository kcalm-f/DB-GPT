# 升级到v0.8.1

## 概述

本指南将引导您完成从 **v0.8.0** 升级到 **v0.8.1**。

- **SQLite 用户**：升级前备份元数据数据库。 DB-GPT 服务启动时会自动创建新的元数据表。
- **MySQL 用户**：需要数据库迁移。应用“assets/schema/upgrade/v0_8_1/”下的增量脚本。

## 准备

### 备份您的数据库

:::警告
为了防止数据丢失，**在升级之前务必备份元数据数据库**。选择与您的数据库类型匹配的备份方法，例如 MySQL 的“mysqldump”或 SQLite 的文件副本。
:::

### 查看升级脚本

v0.8.1 升级脚本保存在 [`assets/schema/upgrade/v0_8_1/`](https://github.com/eosphoros-ai/DB-GPT/tree/main/assets/schema/upgrade/v0_8_1) 下的存储库中：

|脚本 |目的|
| ---| ---|
| `upgrade_to_v0.8.1.sql` |用于升级现有 v0.8.0 MySQL 元数据数据库的增量脚本。 |
| `v0.8.1.sql` |适用于全新安装的完整 v0.8.1 架构。 |

从 v0.8.0 升级时使用 `upgrade_to_v0.8.1.sql`。不要针对现有数据库运行完整架构文件。

## 升级

### 步骤 1 — 停止 DB-GPT 服务

使用与启动 DB-GPT 服务相同的方法停止正在运行的 DB-GPT 服务。

### 步骤 2 — 升级数据库

v0.8.1 元数据升级包含一列更改和三个新元数据表。

**列更改**

|对象|改变 |描述 |
| ---| ---| ---|
| `gpts_messages.content` |将列类型更改为“长文本” |允许存储更长的代理消息和执行跟踪。 |

**新表**

|表|描述 |
| ---| ---|
| `连接器实例` |存储 MCP 连接器实例、加密凭据、传输/额外配置、生命周期状态、用户名和系统代码。 |
| `dbgpt_serve_scheduled_task` |存储计划任务定义、cron 表达式、启用状态以及聊天重放任务的冻结对话快照。 |
| `dbgpt_serve_scheduled_run` |存储计划任务运行历史记录，包括状态、开始/结束时间、结果摘要、错误消息和生成的对话 ID。 |

对于 MySQL 元数据数据库，运行增量升级脚本：
```bash
mysql -u <user> -p dbgpt < assets/schema/upgrade/v0_8_1/upgrade_to_v0.8.1.sql
```
如果您的元数据数据库名称不是“dbgpt”，请将“dbgpt”替换为您的实际数据库名称或在正确的数据库上下文中执行 SQL 文件。

:::提示
增量脚本以“USE dbgpt;”开头。如果您的元数据数据库使用不同的名称，请在运行脚本之前更新该行，或者在 MySQL 会话中显式选择目标数据库。
:::

### 步骤 3 — 安装依赖项

根据您的安装方法安装或更新依赖项。如果您使用默认设置从源安装，请运行：
```bash
uv sync --all-packages
```
如果您计划使用 v0.8.1 中引入的新可选集成，请根据需要安装相应的附加组件：
```bash
# LiteLLM proxy provider
uv sync --all-packages --extra "proxy_litellm"

# Qdrant vector store
uv sync --all-packages --extra "storage_qdrant"

# Valkey cache / vector store
uv sync --all-packages --extra "storage_valkey"
```
### 步骤 4 — 启动 DB-GPT 服务

使用您首选的启动方法重新启动 DB-GPT 服务。

启动后，验证：

- 仍然可以加载现有对话。
- MCP 连接器页面可以列出、激活和测试连接器。
- 可以从已完成的对话中保存计划任务并显示运行历史记录。
- 长时间运行的 ReAct/Data Agent 对话显示任务计划和上下文使用状态。

## 回滚注意事项

如果升级失败，请停止DB-GPT服务，并从升级前的备份中恢复元数据数据库。由于 v0.8.1 更改了元数据架构，因此不要继续运行部分升级的数据库。
