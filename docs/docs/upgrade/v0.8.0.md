# 升级到v0.8.0

## 概述

本指南将引导您完成从 **v0.7.x** 升级到 **v0.8.0**。

- **SQLite 用户**：不需要数据库迁移。
- **MySQL 用户**：需要数据库迁移 - 请参阅下面的[升级数据库](#upgrade-database)。

## 准备

### 备份您的数据库

:::警告
为了防止数据丢失，**在执行升级之前务必备份数据库**。选择与您的数据库类型匹配的备份方法（例如，MySQL 为“mysqldump”，SQLite 为文件复制）。
:::

## 升级

### 步骤 1 — 停止 DB-GPT 服务

使用与启动 DB-GPT 服务相同的方法停止正在运行的 DB-GPT 服务。

### 步骤 2 — 升级数据库

对 MySQL 数据库执行以下 SQL 语句以应用 v0.8.0 架构更改。

:::提示
所有升级 SQL 脚本都保存在存储库中的 [`assets/schema/upgrade/`](https://github.com/eosphoros-ai/DB-GPT/tree/main/assets/schema/upgrade) 下。
:::

**新表**

|表|描述 |
|--------|-------------|
| `分享链接` |存储对话共享链接令牌，使用户能够通过唯一的 URL 共享对话。 |
```sql
USE dbgpt;

-- share_links: Store conversation share link tokens
CREATE TABLE `share_links` (
  `id` int NOT NULL AUTO_INCREMENT COMMENT 'Primary key',
  `token` varchar(64) NOT NULL COMMENT 'Unique random share token',
  `conv_uid` varchar(255) NOT NULL COMMENT 'The conversation uid being shared',
  `created_by` varchar(255) DEFAULT NULL COMMENT 'User who created the share link',
  `gmt_created` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT 'Creation time',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_share_token` (`token`),
  KEY `ix_share_links_token` (`token`),
  KEY `ix_share_links_conv_uid` (`conv_uid`)
) ENGINE=InnoDB AUTO_INCREMENT=1 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='Conversation share link table';
```
### 步骤 3 — 安装依赖项

根据您的安装方法安装或更新依赖项。如果您使用默认设置从源安装，请运行：
```bash
uv sync --all-packages
```
### 步骤 4 — 启动 DB-GPT 服务

使用您首选的启动方法重新启动 DB-GPT 服务。验证服务是否正常运行以及新的共享链接功能是否可用。
