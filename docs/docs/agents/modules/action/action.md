# # #行动目标

客服代表希望通过行动实现什么？

1.任务完成：完成具体任务，在软件开发中编写函数，
并在游戏中做一个铁挑。

2.沟通：与其他客服代表沟通。

3.环境探索：探索陌生环境，拓展感知
并在探索和利用之间取得平衡。

# # #动作制作

操作是如何生成的？
1.通过记忆回忆进行操作。 在此策略中，动作由 
根据当前任务从代理内存中提取信息，任务 
提取的内存用作触发代理操作的提示。
2.通过以下计划采取行动。 在此策略中，客服代表按照其预先生成的计划采取行动。

# # #行动空间

有哪些可用的操作？
操作空间是指座席可以执行的一组可能的操作。
一般来说，我们可以将这些动作大致分为两类： 
（ 1 ）外部工具和（ 2 ） LLM的内部知识。

# # #行动影响

这些行动的后果是什么？
行动影响是指行动的后果。 事实上，行动影响可能包括许多情况。

以下是一些行动影响示例：
1.环境变化：智能体可以通过动作直接改变环境状态，例如 
例如移动位置、收集物品和建造建筑物。
2.更改内部状态：专员采取的行动也可以更改专员本身，
包括更新记忆、制定新计划、获取新知识等。
3.触发新动作：在任务完成过程中，一个客服代表的动作可以被另一个客服代表的动作触发。
