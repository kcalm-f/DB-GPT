# 3.1 Http触发

## 简介

在本章中，我们将开始关注如何使用AWEL来开发网络程序。 
首先，我们将创建一个简单的 HTTP 触发器，用于接收请求并返回响应。

`HttpTrigger` 是一个特殊的 `InputOperator`。

## 安装

我们已经在中创建了一个名为“awel-tutorial”的项目 
[上一章](/docs/awel/awel_tutorial/getting_started/1.1_hello_world#creating-a-project)
并添加了“dbgpt”依赖项。

## 第一个 HTTP 触发器

在`awel_tutorial`目录中创建一个名为`frist_http_trigger_hello.py`的新文件，并添加以下代码：
```python
from dbgpt.core.awel import DAG, HttpTrigger, MapOperator, setup_dev_environment

with DAG("awel_hello_world") as dag:
    trigger_task = HttpTrigger(endpoint="/awel_tutorial/hello_world")
    task = MapOperator(map_function=lambda x: f"Hello, world!")
    trigger_task >> task  
    
setup_dev_environment([dag], port=5555)
```
并运行以下命令来执行代码：
```bash
poetry run python awel_tutorial/frist_http_trigger_hello.py
```
主要输出应该如下所示：
```plaintext
2024-03-03 16:26:57 | INFO | dbgpt.core.awel.trigger.http_trigger | Mount http trigger success, path: /api/v1/awel/trigger/awel_tutorial/hello_world
2024-03-03 16:26:57 | INFO | dbgpt.core.awel.trigger.trigger_manager | Include router <fastapi.routing.APIRouter object at 0x10ed64e50> to prefix path /api/v1/awel/trigger
INFO:     Started server process [69774]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://127.0.0.1:5555 (Press CTRL+C to quit)
```
在 AWEL 中，默认情况下所有 HTTP 端点都以“/api/v1/awel/trigger”为前缀。

现在，打开一个新终端并运行以下命令向服务器发送请求：
```bash
curl -X GET http://127.0.0.1:5555/api/v1/awel/trigger/awel_tutorial/hello_world
```
输出应如下所示：
```plaintext
"Hello, world!"
```
恭喜！您已经创建了第一个 HTTP 触发器。

然后您可以通过在终端中按“Ctrl+C”来停止服务器。
 
## 它是如何工作的

在上面的代码中，我们创建了一个“HttpTrigger”运算符和一个“MapOperator”运算符。
`HttpTrigger`定义了HTTP请求的端点，以及请求的方法 
默认为“GET”。

`setup_dev_environment`函数用于启动服务器并注册dags，它 
如果 DAG 中存在“HttpTrigger”运算符并监听，将阻塞主线程 
默认在 5555 端口上。

当服务器收到请求时，会调用`MapOperator`算子进行处理 
请求并返回结果。

在`HttpTrigger`中，您可以配置端点、方法、请求体、响应体、 
响应状态码等
