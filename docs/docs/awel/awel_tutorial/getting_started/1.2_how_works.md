# 1.2 AWEL 的工作原理

## 简介

让我们再看一下上一节中的 DAG：
```python
import asyncio

from dbgpt.core.awel import DAG, MapOperator, InputOperator, SimpleCallDataInputSource

with DAG("awel_hello_world") as dag:
    input_task = InputOperator(
        input_source=SimpleCallDataInputSource()
    )
    task = MapOperator(map_function=lambda x: print(f"Hello, {x}!"))
    input_task >> task

dag.visualize_dag()
asyncio.run(task.call(call_data="world"))
```
其中代码包含一些新概念：“DAG”、“Operator”和“Task”。

- `DAG`：`DAG` 是一个表示**有向无环图**的类。它用于 
定义任务的结构及其依赖性。
- `Operator`：`InputOperator` 和 `MapOperator` 是运算符的示例。操作员 
是 DAG 中的一个节点。它可以是数据源、转换或数据汇。 
在此示例中，“InputOperator”是数据源，“MapOperator”是数据源 
转变。
- `Task`：任务是一个操作符的实例，它是一个动态的概念。
- `Runner`：Runner 用于执行 DAG 中的任务。当我们 
调用 `task.call(call_data="world")`，我们使用 runner 来执行任务。的 
`DefaultWorkflowRunner` 在同一进程中运行您的任务。还有 
`RayWorkflowRunner` 在 Ray 集群中运行你的任务（社区尚未实现） 
版本）。

## 有向无环图

### 什么是 DAG？

有向无环图 (DAG) 是具有一组顶点和一组 
有向边。边从一个顶点指向另一个顶点，并且没有 
图中的循环。在 AWEL 的上下文中，顶点是运算符，并且 
边是运算符之间的依赖关系。


## 运算符

### 什么是操作员？

算子是 DAG 中的一个节点。它可以是数据源、转换或 
致电 LLM 服务。在 AWEL 的上下文中，运算符是继承的类 
来自“dbgpt.core.awel.BaseOperator”类。 

根据输出数据的类型，运算符有两类： 
**流媒体运营商**和**非流媒体运营商**。

### 基本运算符

有一些基本运算符用于构建更复杂的运算符。

- `InputOperator`：这个**非流式**运算符用于从输入获取数据 
来源。 
- `MapOperator`：这个**非流式**运算符用于将函数应用于 
输入数据并返回转换后的数据。
- `BranchOperator`：这个**非流式**运算符用于决定运行哪条路径 
基于输入数据。
- `JoinOperator`：这个**非流式**运算符用于连接多个数据
 路径合并为单一路径。
- `StreamifyAbsOperator`：这个 **streaming** 运算符用于转换 
非流式运算符到流式运算符。
- `UnstreamifyAbsOperator`：这个**非流**运算符用于转换 
流数据到非流数据。
- `TransformStreamAbsOperator`：这个**流**运算符用于转换 
流数据到另一个流数据。
- `ReduceStreamOperator`：这个**非流**运算符用于减少 
流式传输到非流式数据。
- `TriggerOperator`：这个**非流式**运算符用于触发任务。 
这是一个特殊的“InputOperator”


### 高级操作员

- `RequestBuilderOperator`：这个**非流式**运算符用于构建模型 
来自输入数据的请求。
- `LLMOperator`：这个**非流**运算符用于调用 LLM 服务。
- `StreamingLLMOperator`：这个 **streaming** 运算符用于调用 LLM 服务并 
期待流响应。
- `LLMBranchOperator`：这个**非流式**运算符用于决定选择哪条路径 
根据输入数据运行。
- `OpenAIStreamingOutputOperator`：这个 **streaming** 运算符用于转换模型 
输出到与OpenAI兼容的流数据。
- `ChatHistoryPromptComposerOperator`：这个**非流**操作符用于构建 
编写聊天历史提示的高级任务。


## 任务

### 什么是任务？

任务是操作符的一个实例。它是无状态的设计，意味着任务 
可以使用不同的输入数据执行多次。

每个任务可以从父任务接收多个输入数据，并返回一个 
向子任务输出数据。

## 跑步者

### 什么是跑步者？

运行程序是用于执行 DAG 中任务的类。当我们调用任务时
通过 `task.call(call_data="world")`，我们使用 runner 来执行任务。它将 
触发该任务的所有父任务，然后执行该任务。

`DefaultWorkflowRunner` 在同一进程中运行您的任务。还有 
`RayWorkflowRunner` 在 Ray 集群中运行你的任务（社区尚未实现） 
版本）。此外，您还可以实现自己的运行程序来运行您自己的任务 
环境。
