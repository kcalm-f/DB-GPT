# DB-GPT V0.6.0，定义AI原生数据应用新标准。

## 简介

DB-GPT 是一个带有 AWEL 和代理的开源 AI 原生数据应用程序开发框架。在V0.6.0版本中，我们进一步围绕大模型提供灵活可扩展的AI原生数据应用管理和开发能力，可以帮助企业快速构建和部署智能AI数据应用，通过智能数据分析、洞察和决策，实现企业数字化转型和业务增长


### V0.6.0版本主要增加和增强了以下核心功能

- AWEL协议升级2.0，支持更复杂的编排

- 支持数据应用的创建和生命周期管理，支持多种应用构建模式，如：多Agent自动规划模式、任务流编排模式、单Agent模式、原生应用模式

- GraphRAG支持图社区汇总和混合检索，图索引成本相比微软GraphRAG降低50%。

- 支持多种Agent Memories，如感知记忆、短期记忆、长期记忆、混合记忆等。

- 支持意图识别和提示管理，新增对Text2NLU和Text2GQL微调的支持

- GPT-Vis前端可视化升级，支持更丰富的可视化图表

## 特点

**AWEL协议升级2.0支持更复杂的编排，优化前端可视化和交互能力。**

AWEL（Agentic Workflow Expression Language）是一种专门为大型模型应用开发而设计的基于代理的工作流表达语言，提供强大的功能和灵活性。通过AWEL API，开发者可以专注于大模型应用逻辑开发，而无需关注繁琐的模型、环境等细节。在AWEL2.0中，我们支持更复杂的编排和可视化

**GraphRAG支持图社区汇总和混合检索。**

图构建和检索性能相比社区解决方案有明显优势，并且支持炫酷的可视化。 GraphRAG是一个基于知识图谱的增强型检索生成系统。通过知识图谱的构建和检索，进一步增强检索的准确性和召回的稳定性，同时减少大模型的错觉，增强领域应用的效果。 DB-GPT结合TuGraph构建高效的检索增强生成能力

<p对齐=“中心”>
  <img src={'/img/app/graph_rag_pipeline_v0.6.png'} width="800px" />
</p>

DB-GPT 0.6.0版本在DB-GPT 0.5.6版本推出的集矢量索引、图索引、全文索引为一体的通用RAG框架的基础上，增强了图索引（GraphRAG）的能力，支持图社区汇总和混合检索。能力。在新版本中，我们引入了TuGraph内置的Leiden社区发现算法，结合大模型提取社区子图摘要，最终利用社区摘要的相似性召回来应对广义提问场景，即QFS（Query Focused Summarization）。问题。此外，在知识抽取阶段，我们将原来的三元组抽取升级为点边缘信息汇总的图抽取，并通过文本块历史优化跨文本块关联信息抽取，进一步提升知识图谱的信息密度。


基于上述设计，我们使用TuGraph社区提供的开源知识图谱语料库（OSGraph）以及DB-GPT和TuGraph的产品介绍材料（总计约43k token），并与微软的GraphRAG系统进行了对比测试。最后，DB-GPT 只消耗了 50% 的 token 开销，就生成了相同规模的知识图谱。并且在问答测试质量相当的前提下，全局搜索性能得到了显着提升。
