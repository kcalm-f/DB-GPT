# DB-GPT V0.7.0、MCP + DeepSeek R1：为LLM申请带来更多可能

**DB-GPT V0.7.0 发布：MCP 协议支持、DeepSeek R1 模型集成、完整架构升级、GraphRAG 检索链增强等..**

“DB-GPT”是一个带有 AWEL 和代理的开源 AI 本机数据应用程序开发框架。在‘V0.7.0’版本中，我们对‘DB-GPT’模块包进行了重组，对原有模块进行了拆分，重构了整个框架配置体系，为围绕大模型构建AI原生数据应用提供了更清晰、更灵活、更可扩展的管理和开发能力。

## V0.7.0版本主要增加和增强了以下核心功能

🍀 **支持** `MCP（模型上下文协议）` **协议。**

🍀 **集成** `DeepSeek R1`、`QWQ` **推理模型，全原创** `DB-GPT` **聊天场景现已涵盖深度思考能力。**

🍀 `GraphRAG` **检索链增强：支持“Vector”和“Intent Recognition+Text2GQL”图形检索器。**

🍀 `DB-GPT` **模块包重组，原来** `dbgpt` **包拆分成** `dbgpt-core`、`dbgpt-ext`、`dbgpt-serve`、`dbgpt-client`、`dbgpt-acclerator`、`dbgpt-app`。

🍀 **重构** `DB-GPT` **配置系统，配置文件改为“**`.toml`**”格式，废除了原有的** `.env` **配置逻辑。**

### 3. `GraphRAG` **检索链增强：支持“Vector”和“Intent Recognition+Text2GQL”图形检索器。**

+ **“矢量”图形检索器**

在知识图谱构建过程中，对所有节点添加向量，并建立边和索引。查询时，将问题向量化，通过TuGraph-DB内置的向量索引能力，基于HNSW算法，查询topk相关的节点和边。与关键词图检索相比，它可以识别更多模糊问题。

<p对齐=“中心”>
  <img src={'/img/v070/graphrag_1.png'} width="800px"/>
</p>

配置示例：
```toml
[rag.storage.graph]
type = "TuGraph"
host="127.0.0.1"
port=7687
username="admin"
password="73@TuGraph"

enable_summary="True"
triplet_graph_enabled="True"
document_graph_enabled="True"

# Vector graph retrieval configuration items
enable_similarity_search="True"
knowledge_graph_embedding_batch_size=20
similarity_search_topk=5
extract_score_threshold=0.7
```
+ **“意图识别+Text2GQL”图检索器**

通过意图识别模块重写问题，提取真实意图以及涉及的实体和关系，然后使用Text2GQL模型翻译成GQL语句进行直接查询。可以进行更精准的图形查询，并显示相应的查询语句。除了调用大模型API服务外，您还可以使用ollama调用本地Text2GQL模型。


<p对齐=“中心”>
  <img src={'/img/v070/graphrag_2.png'} width="800px"/>
</p>


配置示例：
```toml
[rag.storage.graph]
type = "TuGraph"
host="127.0.0.1"
port=7687
username="admin"
password="73@TuGraph"

enable_summary="True"
triplet_graph_enabled="True"
document_graph_enabled="True"

# Intent Recognition+Text2GQL graph retrieval configuration items
enable_text_search="True"

# Use Ollama to deploy independent text2gql model, enable the following configuration items
# text2gql_model_enabled="True"
# text2gql_model_name="tugraph/CodeLlama-7b-Cypher-hf:latest"
```
### 4.`DB-GPT`模块包重构

原始 `dbgpt` 包分为 `dbgpt-core`、`dbgpt-ext`、`dbgpt-serve`、`dbgpt-client`、`dbgpt-acclerator`、`dbgpt-app`

随着dbgpt的逐步发展，服务模块增多，功能回归测试变得困难，兼容性问题也更加频繁。因此，对原有的dbgpt内容进行了模块化：

+ **dbgpt-core**：主要负责dbgpt的awel、model、agent、rag、storage、datasource等核心模块接口定义，发布Python SDK。
+ **dbgpt-ext**：主要负责实现dbgpt扩展内容，包括数据源扩展、向量存储、图存储扩展、模型访问扩展，方便社区开发者快速使用和扩展新模块内容，发布Python SDK。
+ **dbgpt-serve**：主要为dbgpt各模块的原子化服务提供Restful接口，方便社区用户快速集成。目前还没有发布Python SDK。
+ **dbgpt-app**：主要负责`App`、`ChatData`、`ChatKnowledge`、`ChatExcel`、`Dashboard`等业务场景实现，无Python SDK。
+ **dbgpt-client**：提供统一的Python SDK客户端进行集成。
+ **dbgpt-accelerator：** 模型推理加速模块，包括对不同版本（不同torch版本等）、平台（Windows、MacOS、Linux）、硬件环境（CPU、CUDA、ROCM）、推理框架（vLLM、llama.cpp）、量化方法（AWQ、bitsandbytes、GPTQ）以及其他加速模块（accelerate、flash-attn）的兼容性和适配，提供跨平台、其他 DB-GPT 模块可按需安装的底层环境。


### 5.重构DB-GPT配置系统

新的配置文件采用“.toml”格式，废除了原来的“.env”配置逻辑，每个模块可以有自己的配置类，并自动生成前端配置页面。

快速使用参考：[http://docs.dbgpt.cn/docs/next/quickstart](http://docs.dbgpt.cn/docs/next/quickstart)

所有配置参考：[http://docs.dbgpt.cn/docs/next/config-reference/app/config_chatdashboardconfig_2480d0](http://docs.dbgpt.cn/docs/next/config-reference/app/config_chatdashboardconfig_2480d0)
```toml
[system]
# Load language from environment variable(It is set by the hook)
language = "${env:DBGPT_LANG:-zh}"
api_keys = []
encrypt_key = "your_secret_key"

# Server Configurations
[service.web]
host = "0.0.0.0"
port = 5670

[service.web.database]
type = "sqlite"
path = "pilot/meta_data/dbgpt.db"
[service.model.worker]
host = "127.0.0.1"

[rag.storage]
[rag.storage.vector]
type = "chroma"
persist_path = "pilot/data"

# Model Configurations
[models]
[[models.llms]]
name = "deepseek-reasoner"
# name = "deepseek-chat"
provider = "proxy/deepseek"
api_key = "your_deepseek_api_key"
```
### 6. **支持S3、OSS存储**

DB-GPT统一存储扩展OSS和S3实现，其中S3实现支持大部分兼容S3协议的云存储。 DB-GPT知识库原始文件、Chat Excel相关中间文件、AWEL Flow节点参数文件等均支持云存储。

配置示例：
```toml
[[serves]]
type = "file"
# Default backend for file server
default_backend = "s3"

[[serves.backends]]
type = "oss"
endpoint = "https://oss-cn-beijing.aliyuncs.com"
region = "oss-cn-beijing"
access_key_id = "${env:OSS_ACCESS_KEY_ID}"
access_key_secret = "${env:OSS_ACCESS_KEY_SECRET}"
fixed_bucket = "{your_bucket_name}"

[[serves.backends]]
# Use Tencent COS s3 compatible API as the file server
type = "s3"
endpoint = "https://cos.ap-beijing.myqcloud.com"
region = "ap-beijing"
access_key_id = "${env:COS_SECRETID}"
access_key_secret = "${env:COS_SECRETKEY}"
fixed_bucket = "{your_bucket_name}
```
详细配置说明请参考：[http://docs.dbgpt.cn/docs/next/config-reference/utils/config_s3storageconfig_f0cdc9](http://docs.dbgpt.cn/docs/next/config-reference/utils/config_s3storageconfig_f0cdc9)

### 15. **DB-GPT API 与 OpenAI SDK 兼容**
```python
from openai import OpenAI

DBGPT_API_KEY = "dbgpt"

client = OpenAI(
    api_key=DBGPT_API_KEY,
    base_url="http://localhost:5670/api/v2",
)

messages = [
    {
        "role": "user",
        "content": "Hello, how are you?",
    },
]

has_thinking = False
reasoning_content = ""
for chunk in client.chat.completions.create(
    model="deepseek-chat",
    messages=messages,
    extra_body={
        "chat_mode": "chat_normal",
    },
    stream=True,
    max_tokens=4096,
):
    delta_content = chunk.choices[0].delta.content
    if hasattr(chunk.choices[0].delta, "reasoning_content"):
        reasoning_content = chunk.choices[0].delta.reasoning_content
    if reasoning_content:
        if not has_thinking:
            print("<thinking>", flush=True)
        print(reasoning_content, end="", flush=True)
        has_thinking = True
    if delta_content:
        if has_thinking:
            print("</thinking>", flush=True)
        print(delta_content, end="", flush=True)
        has_thinking = False

```

**升级指南：**

### 1.元数据数据库升级

对于SQLite升级，表结构默认会自动升级。对于MySQL升级，需要手动执行DDL。 `assets/schema/dbgpt.sql` 文件包含当前版本的完整 DDL。具体的版本变更DDL可以在`assets/schema/upgrade`目录中找到。例如，如果您要从 v0.6.3 升级到 v0.7.0，则可以执行以下 DDL：
```bash
mysql -h127.0.0.1 -uroot -p{your_password} < ./assets/schema/upgrade/v0_7_0/upgrade_to_v0.7.0.sql
```
### 2.矢量数据库升级

由于 v0.7.0 中 Chroma 存储的根本变化，0.7.0 版本不支持读取旧版本的内容。请重新导入知识库并刷新数据源。其他矢量存储解决方案不受影响。
